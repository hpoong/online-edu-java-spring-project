


스트레스 테스트 (Stress Test):
    목적: 시스템이 얼마나 많은 부하(스트레스)를 견딜 수 있는지 확인하는 테스트입니다.
    설명: 일반적인 작업량보다 훨씬 많은 부하를 주어, 시스템의 성능 한계나 취약점을 찾는 데 집중합니다. 예를 들어, 웹 서버에 평소보다 더 많은 요청을 보내 서버가 어떻게 반응하는지 테스트합니다.

스파이크 테스트 (Spike Test):
    목적: 시스템이 갑작스럽게 증가하는 부하를 어떻게 처리하는지 확인하는 테스트입니다.
    설명: 짧은 시간 안에 부하가 급격히 증가하는 상황을 시뮬레이션하여, 시스템이 이 상황을 어떻게 버티는지 테스트합니다. 예를 들어, 갑자기 사용자가 급증하는 이벤트가 발생했을 때 시스템이 어떻게 대응하는지 확인합니다.

인더런스 테스트 (Endurance Test):
    목적: 시스템이 오랜 시간 동안 안정적으로 작동하는지를 확인하는 테스트입니다.
    설명: 장기간 동안 지속적으로 부하를 주어 시스템이 시간이 지나면서 성능 저하나 오류가 발생하는지를 확인합니다. 예를 들어, 서버가 몇 시간 또는 며칠 동안 연속적으로 작동할 때 메모리 누수나 성능 저하가 발생하는지를 검사합니다.




시나리오

STRESS 테스트
    500명의 동시 사용자가 초당 50번을 호출하여 분당(5분) 사용자를 50씩 늘려 서버의 지표를 확인
Endurance 테스트
    100명의 동시 사용자가 초당 100번을 호출하였을때 10분동안 서버의 지표를 확인
PEAK 테스트
    100명의 동시 사용자가 초당 50번씩 호출하다 1분에 1000명으로 사용자를 한번에 늘려 서버의 지표를 확인
